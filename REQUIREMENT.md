# 트라이얼 위크 과제 : URL Shortener

## 1. 과제 개요

이 과제는 긴 웹 주소(URL)를 짧은 형태로 변환하는 단축 URL 서비스(URL Shortener)를 직접 설계하고 구현하는 것입니다.

Bitly, TinyURL 등에서 볼 수 있는 익숙한 형태이며, 단축 URL을 통해 원본 URL로 쉽게 접근할 수 있도록 하는 서비스 입니다.

## 2. 요구사항

### **1) 단축 URL 생성**

- 사용자가 원본 URL을 입력하면 단축 URL을 생성
    - 예: "https://example.com/products/1234567890" → "https://sho.rt/zb2x4"
- 단축 URL에는 **shortCode**가 포함되어야 함
    - shortCode 포맷: 숫자 및 영문 대소문자로 구성된 5~10자리 문자열
- 다양한 URL에 대해 5~10자리 shortCode 고유값 보장을 위한 처리 필요
    - 충돌 발생한 경우 shortCode 고유값 보장을 위한 처리 필요
- 사용자가 원하면, 원본 URL의 성격을 최대한 반영할 수 있는 단축 URL 생성 가능
    - 예: “https://app.bangbangcapital.com/events/2025/christmas” → “https://sho.rt/25chs” or "https://sho.rt/bbcpt"
- 이미 같은 URL이 존재하면 기존 단축 URL을 반환

### **2) 단축 URL 조회**

- 단축 URL 접근 시 원본 URL로 리다이렉트
- 조회 시 방문 횟수 증가 (방문 횟수는 통계 목적)
- 특정 단축 URL이 반복적으로 많이 조회되는 상황이 발생할 수 있음
    - 예: SNS에서 단축 URL이 공유되어 동시에 많은 사용자가 접근
    - 예: 뉴스 기사, 이벤트 페이지, 광고 링크 등에서 단축 URL이 짧은 시간 내에 집중 조회
- 캐시 사용 가능하나, 최대 크기는 100개로 제한
- 캐시 정책과 eviction 방식은 자유롭게 설계 가능
